# CUDA Cache Bandwidth Measurement for A100 GPU

这个程序用于测量A100 GPU的L1和L2 Cache带宽，以及全局内存带宽。

## 功能特性

- **L1 Cache测试**: 使用共享内存和较小的数据块来测量L1 Cache性能
- **L2 Cache测试**: 使用全局内存的跨步访问模式来测量L2 Cache性能
- **全局内存测试**: 使用顺序访问模式来测量全局内存带宽
- **GPU信息显示**: 自动检测并显示GPU的详细规格信息

## 编译要求

- CUDA Toolkit 11.0或更高版本
- 支持CUDA的GPU（推荐A100或类似架构）
- GCC或Clang编译器

## 程序说明

### L1 Cache测试
- 使用64KB数据块，适合L1 Cache大小
- 利用共享内存进行快速访问
- 包含读写操作以获得完整带宽测量

### L2 Cache测试
- 使用8MB数据块，适合L2 Cache大小
- 采用64字节跨步访问模式
- 模拟L2 Cache的典型访问模式

### 全局内存测试
- 使用1GB数据块
- 采用顺序访问模式最大化内存带宽
- 测量理论最大内存带宽

## 输出说明

程序会输出以下信息：

1. **GPU信息**: 设备名称、计算能力、内存大小、Cache大小等
2. **L1 Cache测试结果**: 时间、带宽、总数据量
3. **L2 Cache测试结果**: 时间、带宽、总数据量
4. **全局内存测试结果**: 时间、带宽、总数据量

## 预期结果

对于A100 GPU，预期结果大致如下：

- **L1 Cache带宽**: 1000-2000 GB/s
- **L2 Cache带宽**: 500-1000 GB/s
- **全局内存带宽**: 1500-2000 GB/s

注意：实际结果可能因GPU配置、驱动程序版本和系统负载而异。

## 技术细节

### 内存访问模式

1. **L1 Cache测试**:
   - 使用共享内存作为L1 Cache的代理
   - 数据块大小: 64KB
   - 访问模式: 块内顺序访问

2. **L2 Cache测试**:
   - 使用全局内存的跨步访问
   - 数据块大小: 8MB
   - 跨步大小: 64字节（适合L2 Cache行大小）

3. **全局内存测试**:
   - 使用顺序访问模式
   - 数据块大小: 1GB
   - 最大化内存带宽

### 带宽计算

带宽计算公式：
```
带宽 = (总字节数 × 迭代次数 × 2) / (时间 × 1024³)
```

其中：
- 总字节数：测试的数据大小
- 迭代次数：重复测试的次数
- ×2：包含读写操作
- 时间：以秒为单位

## 故障排除

### 常见问题

1. **编译错误**: 确保CUDA Toolkit已正确安装
2. **运行时错误**: 检查GPU是否支持CUDA
3. **内存不足**: 减少BUFFER_SIZE常量值

### 性能优化建议

1. 确保GPU处于最佳性能模式
2. 关闭其他GPU应用程序
3. 使用最新的CUDA驱动程序
4. 在专用测试环境中运行
