# CUDA Cache Bandwidth Test for A100 GPU

这个程序用于测量NVIDIA A100 GPU的L1和L2 Cache带宽性能。

## 功能特性

- **L1 Cache带宽测试**: 使用共享内存测量L1 Cache的读取带宽
- **L2 Cache带宽测试**: 使用全局内存访问模式测量L2 Cache的读取带宽  
- **写带宽测试**: 测量内存写入带宽
- **GPU设备信息**: 显示详细的GPU硬件信息
- **高精度计时**: 使用C++11 chrono库进行微秒级计时

## 系统要求

- NVIDIA GPU (推荐A100，但也支持其他Compute Capability 8.0+的GPU)
- CUDA Toolkit 11.0或更高版本
- C++11兼容的编译器
- Linux操作系统

## 编译和运行

### 1. 检查CUDA环境
```bash
make check-cuda
```

### 2. 编译程序
```bash
make
```

### 3. 运行测试
```bash
make run
# 或者直接运行
./cache_bandwidth_test
```

### 4. 清理编译文件
```bash
make clean
```

## 程序输出说明

程序会输出以下信息：

1. **GPU设备信息**
   - 设备名称和计算能力
   - 全局内存大小
   - 共享内存大小
   - L2缓存大小
   - 内存时钟频率和带宽
   - 多处理器数量

2. **L1 Cache带宽测试**
   - 数据大小: 32KB (适合L1 Cache)
   - 测试方法: 使用共享内存进行密集读取操作
   - 预期结果: A100约为15-20 TB/s

3. **L2 Cache带宽测试**
   - 数据大小: 4MB (超过L1但适合L2 Cache)
   - 测试方法: 使用stride访问模式
   - 预期结果: A100约为3-7 TB/s

4. **写带宽测试**
   - 数据大小: 2MB
   - 测试方法: 连续写入操作
   - 预期结果: 取决于内存子系统性能

## 测试原理

### L1 Cache测试
- 使用共享内存模拟L1 Cache行为
- 每个线程块将数据加载到共享内存
- 执行大量读取操作来测量带宽
- 共享内存访问延迟极低，带宽极高

### L2 Cache测试  
- 使用全局内存访问，数据大小超过L1但适合L2
- 采用stride访问模式来利用L2 Cache
- 多次重复访问相同数据以确保缓存命中
- 测量L2 Cache的有效带宽

### 写带宽测试
- 测量内存写入性能
- 使用不同的stride模式避免bank冲突
- 评估写入子系统的性能

## A100 GPU预期性能

根据NVIDIA官方规格，A100 GPU的理论性能：

- **L1 Cache**: 每个SM约19.5 MB，带宽可达15+ TB/s
- **L2 Cache**: 40MB，带宽约3-7 TB/s  
- **HBM2e内存**: 1555 GB/s (40GB版本) 或 2039 GB/s (80GB版本)

## 注意事项

1. **编译架构**: Makefile中使用`-arch=sm_80`针对A100优化，其他GPU可能需要调整
2. **内存大小**: 测试使用的数据大小经过优化以适合不同级别的缓存
3. **预热**: 每个测试都包含预热阶段以确保准确的性能测量
4. **防优化**: 使用结果防止编译器优化掉测试代码

## 故障排除

### 编译错误
- 确保安装了CUDA Toolkit: `nvcc --version`
- 检查GPU计算能力: `nvidia-smi`
- 调整Makefile中的架构标志

### 运行时错误
- 检查CUDA驱动: `nvidia-smi`
- 确保有足够的GPU内存
- 检查CUDA运行时版本兼容性

### 性能异常
- 确保GPU处于性能模式而非节能模式
- 检查GPU温度和频率
- 关闭其他GPU应用程序

## 扩展和自定义

可以通过修改以下参数来自定义测试：

- `data_size`: 调整测试数据大小
- `iterations`: 修改测试迭代次数  
- `block_size`: 改变CUDA块大小
- 访问模式: 修改stride和索引计算方式
