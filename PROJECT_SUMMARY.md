# CUDA A100 GPU Cache带宽测试程序

## 项目概述

本项目提供了一个完整的CUDA程序，用于测量NVIDIA A100 GPU的L1和L2 Cache带宽性能。程序包含GPU版本和CPU模拟版本，确保在没有CUDA环境的情况下也能演示程序逻辑。

## 文件结构

```
├── cache_bandwidth_test.cu          # 主要的CUDA GPU测试程序
├── cache_bandwidth_test_cpu.cpp     # CPU模拟版本（用于演示）
├── Makefile                         # 自动化编译脚本
├── install_cuda.sh                  # CUDA安装助手脚本
├── README_cache_test.md            # 详细使用说明
└── PROJECT_SUMMARY.md              # 项目总结（本文件）
```

## 核心功能

### 1. GPU版本 (`cache_bandwidth_test.cu`)

**L1 Cache带宽测试**
- 使用共享内存模拟L1 Cache行为
- 数据大小：32KB（适合L1 Cache）
- 测试方法：密集的共享内存读取操作
- 预期性能：A100约15-20 TB/s

**L2 Cache带宽测试**
- 使用全局内存访问模式
- 数据大小：4MB（超过L1但适合L2）
- 测试方法：stride访问模式利用L2 Cache
- 预期性能：A100约3-7 TB/s

**写带宽测试**
- 测量内存写入性能
- 数据大小：2MB
- 测试方法：连续写入操作
- 评估写入子系统性能

### 2. CPU模拟版本 (`cache_bandwidth_test_cpu.cpp`)

- 模拟GPU内核的访问模式
- 用于演示程序结构和逻辑
- 在没有CUDA环境时提供功能展示
- 显示A100 GPU的理论性能对比

## 技术特点

### 高精度测量
- 使用C++11 chrono库进行微秒级计时
- 包含预热阶段确保准确测量
- 防止编译器优化影响结果

### 内存访问模式优化
- L1测试：利用共享内存的高带宽特性
- L2测试：使用stride访问模式最大化缓存利用率
- 写测试：避免bank冲突的访问模式

### 错误处理
- 完整的CUDA错误检查宏
- 设备兼容性检查
- 内存分配失败处理

### 自动化构建
- 智能检测CUDA环境
- 自动选择GPU或CPU版本编译
- 一键运行和清理

## A100 GPU性能规格

根据NVIDIA官方规格：

| 组件 | 规格 | 带宽 |
|------|------|------|
| L1 Cache | 每SM 19.5MB | 15-20 TB/s |
| L2 Cache | 40MB | 3-7 TB/s |
| HBM2e (40GB) | 1555 GB/s | 1.555 TB/s |
| HBM2e (80GB) | 2039 GB/s | 2.039 TB/s |

## 使用方法

### 快速开始
```bash
# 1. 检查环境
make check-cuda

# 2. 编译（自动选择版本）
make

# 3. 运行测试
make run

# 4. 清理
make clean
```

### CUDA环境安装
```bash
# 使用安装脚本
./install_cuda.sh

# 或者手动安装
sudo apt install nvidia-cuda-toolkit
```

## 测试结果解读

### L1 Cache测试
- 测试共享内存访问性能
- 结果应接近理论峰值带宽
- 受限于内存访问模式和并发度

### L2 Cache测试
- 测试跨SM的缓存性能
- 结果取决于数据局部性
- 反映实际应用中的缓存效率

### 写带宽测试
- 测试内存写入子系统
- 评估数据输出性能
- 对计算密集型应用重要
